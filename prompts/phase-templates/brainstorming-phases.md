# 头脑风暴环节推进模板

## 使用说明

本模板用于头脑风暴方法的环节推进。目标是提高方案数量、质量和多样性。

---

## 经典规则提醒

| 规则 | 说明 |
|------|------|
| 延迟评价 | 不立刻评判好坏 |
| 鼆励数量 | 先追求想法数量 |
| 欢迎异想天开 | 奇怪想法也欢迎 |
| 延展他人想法 | 在别人想法上继续发展 |

---

## Phase 1 - 明确问题

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "头脑风暴：明确问题",
  "message": "【头脑风暴 Phase 1】明确问题\n\n问题/目标：{问题描述}\n\n任务：理解问题背景，明确头脑风暴的目标\n时间盒：5分钟\n\n输出格式要求：\n## {你的Agent名称} - 问题理解\n### 问题复述\n{用自己的话复述问题}\n\n### 关键约束\n- {约束}\n\n### 期望产出\n{期望的想法类型}\n\n请在{时间}内完成输出。"
}
```

### Participant输出模板

```markdown
## {Agent名称} - 问题理解

### 问题复述
{用自己的话复述问题}

### 关键约束
- 约束1: {约束}
- 约束2: {约束}

### 期望产出
- 期望产出{N}个想法
- 期望想法类型：{类型}
```

## 头脑风暴完整流程示例

### 环节推进时间盒建议

| 环节 | 时间盒 | 核心产出 |
|------|--------|----------|
| Phase 1 | 5分钟 | 问题理解确认 |
| Phase 2 | 15分钟 | 想法列表（目标每人5+） |
| Phase 3 | 10分钟 | 投票筛选结果 |
| Phase 4 | 15分钟 | 可执行方案 |
| **总计** | **45分钟** | **创意方案纪要** |

---

## 注意事项

- **Phase 2禁止评价**：任何"好/坏"评价都违规
- **数量优先**：质量在Phase 4解决，Phase 2追求数量
- **延展机制**：鼓励在他人想法上发展
- **防止轮流随口说**：需要有思考时间
- **防止领导定调**：避免第一个发言者定方向
