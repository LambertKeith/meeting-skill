# 通用环节模板

## 使用说明

本模板包含会议开场、环节过渡、会议收尾等通用模板，适用于所有会议类型。

---

## 会议开场模板

### TeamLead开场广播

```json
{
  "to": "*",
  "summary": "会议开场",
  "message": "【会议开场】\n\n会议类型：{类型}\n会议议题：{议题}\n\n议程概览：\n| 序号 | 环节 | 方法 | 时间 |\n|------|------|------|------|\n| 1 | {环节} | {方法} | {时间} |\n\n角色分配：\n- Driver: {TeamLead}\n- Approver: {DecisionMaker}\n- Contributors: {Participants列表}\n\n会议目标：{目标}\n预期产出：{产出}\n\n请确认是否准备就绪。"
}
```

### Participant确认模板

```markdown
## {Agent名称} - 会议确认

### 准备就绪
- 已阅读议程
- 已确认角色

### 补充信息（如有）
- {补充}

### 期望说明（如有）
- {期望}
```

## Shutdown通知模板

### TeamLead Shutdown广播

```json
{
  "to": "*",
  "type": "shutdown_request",
  "message": {
    "type": "shutdown_request",
    "reason": "会议已完成，所有议程和纪要已产出"
  }
}
```

### Participant Shutdown响应

```json
{
  "to": "team-lead",
  "type": "shutdown_response",
  "message": {
    "type": "shutdown_response",
    "request_id": "{request_id}",
    "approve": true
  }
}
```

## 异常处理模板

### 环节阻塞处理

```markdown
## TeamLead - 环节阻塞处理

### 阻塞情况
- 阻塞环节：{环节}
- 阻塞原因：{原因}
- 阻塞参与者：{Agent}

### 处理方案
- 方案1: {方案}
- 方案2: {方案}

### 决策Maker裁决请求
请DecisionMaker裁决阻塞处理方案。
```

### DecisionMaker裁决响应

```markdown
## DecisionMaker - 阻塞裁决

### 裁决结果
- 选择方案：{方案}
- 裁决理由：{理由}

### 执行指示
- {指示}
```

## 检查点模板

### 环节完成检查点

```markdown
## 环节完成检查点

### 环节名称：{环节}

### 检查项
| 检查项 | 状态 | 说明 |
|--------|------|------|
| 所有Participant已输出 | ✅/❌ | {说明} |
| 输出格式符合模板 | ✅/❌ | {说明} |
| TeamLead已汇总 | ✅/❌ | {说明} |
| 共识已形成 | ✅/❌ | {说明} |
| 待决已记录 | ✅/❌ | {说明} |

### 检查结果
- 通过/需补充：{结果}
```
