# 六顶思考帽环节推进模板

## 使用说明

本模板用于六顶思考帽方法的环节推进。通过不同"帽子"代表不同思考角度，让团队在同一时间只做一种类型的思考。

---

## 帽子顺序选择

| 场景 | 推荐顺序 |
|------|----------|
| 问题分析 | 白 → 红 → 黑 → 黄 → 绿 → 蓝 |
| 方案评估 | 白 → 黄 → 黑 → 绿 → 蓝 |
| 创意发散 | 绿 → 黄 → 黑 → 蓝 |

---

## 白帽环节 - 事实数据

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "白帽：事实数据",
  "message": "【白帽】事实数据\n\n思考类型：列出所有已知事实、数据、信息\n时间盒：10分钟\n\n规则：\n- 只陈述事实，不表达观点\n- 区分"已知"和"需要知道"\n- 使用数据支撑\n\n输出格式要求：\n## {你的Agent名称} - 白帽思考\n### 已知事实\n1. {事实}\n\n### 已知数据\n- {数据}\n\n### 需要知道的信息\n- {信息}\n\n请在{时间}内完成输出。"
}
```

### Participant输出模板

```markdown
## {Agent名称} - 白帽思考

### 已知事实
1. {事实1}: {具体描述}
2. {事实2}: {具体描述}
3. {事实3}: {具体描述}

### 已知数据
- {数据项}: {数值/状态}
- {数据项}: {数值/状态}

### 需要知道的信息
- {信息1}: {为什么需要}
- {信息2}: {为什么需要}

### 事实来源
- {事实}: 来源{来源}
```

### TeamLead汇总模板

```markdown
## 六顶思考帽 - 白帽汇总

### 事实汇总表
| 事实 | 提及次数 | 来源 |
|------|----------|------|
| {事实1} | {N} | {来源} |
| {事实2} | {N} | {来源} |

### 数据汇总
| 数据项 | 数值 | 提供者 |
|--------|------|--------|
| {项} | {值} | {Agent} |

### 需要补充的信息
- {信息}: {N}人提及需要
```

## 六顶思考帽完整流程示例

### 环节推进时间盒建议（问题分析顺序）

| 环节 | 时间盒 | 核心产出 |
|------|--------|----------|
| 白帽 | 10分钟 | 事实数据列表 |
| 红帽 | 5分钟 | 情绪感受 |
| 黑帽 | 10分钟 | 风险隐患列表 |
| 黄帽 | 10分钟 | 价值机会列表 |
| 绿帽 | 15分钟 | 新方案列表 |
| 蓝帽 | 10分钟 | 结论、下一步 |
| **总计** | **60分钟** | **完整会议纪要** |

---

## 注意事项

- **同频思考**：所有人同一时间只戴一种帽子
- **红帽限时**：严格控制时间，不延展讨论
- **绿帽不评**：禁止在绿帽环节评价想法好坏
- **蓝帽收尾**：必须形成结论和下一步
