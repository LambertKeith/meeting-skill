# ORID 环节推进模板

## 使用说明

本模板用于ORID焦点讨论法的环节推进。TeamLead使用SendMessage广播环节任务，Participant按模板格式输出。

---

## Phase O - 客观事实

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "环节O：客观事实收集",
  "message": "【环节O】客观事实收集\n\n任务：列出所有客观事实、数据、现象，不掺杂观点\n时间盒：10分钟\n\n输出格式要求：\n## {你的Agent名称} - 客观事实\n### 观察到的事实\n1. ...\n2. ...\n\n### 相关数据\n- ...\n\n### 事件时间线\n- ...\n\n请在{时间}内完成输出。"
}
```

### Participant输出模板

```markdown
## {Agent名称} - 客观事实

### 观察到的事实
1. {具体事实描述}
2. {具体事实描述}
3. {具体事实描述}

### 相关数据
- {数据项}: {数值/状态}
- {数据项}: {数值/状态}

### 事件时间线
- {时间点}: {发生的事件}
- {时间点}: {发生的事件}
```

### TeamLead汇总模板

```markdown
## ORID Phase O 汇总

### 事实汇总表
| 来源 | 关键事实 | 相关数据 |
|------|----------|----------|
| {Agent A} | ... | ... |
| {Agent B} | ... | ... |

### 事件时间线合并
| 时间 | 事件 | 提供者 |
|------|------|--------|
| ... | ... | ... |

### 需要澄清的事实
- {Agent A}与{Agent B}描述不一致的点：...
```

## Phase R - 感受反应

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "环节R：感受反应",
  "message": "【环节R】感受反应\n\n任务：表达对关键事实的感受、情绪、联想\n时间盒：5分钟\n规则：可以表达感受，不争论，不质疑\n\n输出格式要求：\n## {你的Agent名称} - 感受反应\n### 对关键事实的反应\n- 事实{X}令我{感受}，因为{原因}\n\n### 惊讶/担忧/期待\n- ...\n\n请在{时间}内完成输出。"
}
```

```markdown
## {Agent名称} - 感受反应

- {令我惊讶的内容}

### 担忧的点
- {令我担忧的内容}

- {我期待的结果}
```

```markdown
|----------|----------|----------|
```

## Phase I - 理解判断

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "环节I：理解判断",
  "message": "【环节I】理解判断\n\n任务：分析事实的意义、原因、关联，形成洞察\n时间盒：15分钟\n规则：要有依据，不能空谈，联系Phase O的事实\n\n输出格式要求：\n## {你的Agent名称} - 理解判断\n### 核心洞察\n1. {洞察内容}\n\n### 原因分析\n- {原因1}\n- {原因2}\n\n### 意义解读\n- ...\n\n请在{时间}内完成输出。"
}
```

### Participant输出模板

```markdown
## {Agent名称} - 理解判断

### 核心洞察
1. {洞察1}: {基于Phase O的事实{X}，我判断...}
2. {洞察2}: {基于Phase O的事实{Y}，我判断...}

### 原因分析
- 根因可能是：{分析}
- 直接原因：{分析}
- 间接原因：{分析}

### 意义解读
- 这对{方面}意味着{解读}
- 这表明{结论}

### 关联发现
- {事实A}与{事实B}存在关联：{分析}
```

### TeamLead汇总模板

```markdown
## ORID Phase I 汇总

### 共性洞察
| 洞察 | 提及次数 | 支持事实 |
|------|----------|----------|
| {洞察1} | {N} | {事实X} |
| {洞察2} | {N} | {事实Y} |

### 原因分析汇总
- **根因共识**: {N}人认为根因是{原因}
- **分歧点**: {Agent A}认为{原因A}，{Agent B}认为{原因B}

### 待验证判断
- {判断}需要进一步数据验证
```

## Phase D - 决策行动

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "环节D：决策行动",
  "message": "【环节D】决策行动\n\n任务：基于Phase I的洞察，形成结论和可执行项\n时间盒：10分钟\n规则：必须产出可执行项，有明确的Who/What/When\n\n输出格式要求：\n## {你的Agent名称} - 决策行动\n### 结论\n- {结论内容}\n\n### 建议行动项\n| 序号 | 行动 | 负责人 | 时间 |\n|------|------|--------|------|\n| 1 | ... | ... | ... |\n\n### 待决问题\n- ...\n\n请在{时间}内完成输出。"
}
```

### Participant输出模板

```markdown
## {Agent名称} - 决策行动

### 结论
- 结论1: {基于Phase I洞察{X}，建议{结论}}

- {需要后续讨论的问题}
```

## ORID完整流程示例

### 环节推进时间盒建议

| 环节 | 时间盒 | 核心产出 |
|------|--------|----------|
| Phase O | 10分钟 | 事实列表、数据汇总 |
| Phase R | 5分钟 | 情绪分布、共性反应 |
| Phase I | 15分钟 | 洞察汇总、原因分析 |
| Phase D | 10分钟 | 结论、行动项 |
| **总计** | **40分钟** | **完整会议纪要** |

### 推进检查点

1. **Phase O结束后**：确认所有关键事实已列出，无遗漏
2. **Phase R结束后**：确认情绪已表达，不延展争论
3. **Phase I结束后**：确认洞察有事实支撑，形成共识
4. **Phase D结束后**：确认行动项有明确Who/What/When

---

## 注意事项

- **Phase O禁止**：表达观点、推断原因、提出方案
- **Phase R控制**：篇幅不宜过长，避免情绪宣泄
- **Phase I要求**：必须有Phase O事实支撑，不能空谈
- **Phase D必须**：产出可执行项，有明确责任人
