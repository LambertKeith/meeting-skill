# 决策矩阵环节推进模板

## 使用说明

本模板用于决策矩阵方法的环节推进。适用于在多个方案间做理性比较的决策场景。

---

## Phase 1 - 确定评价标准

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "环节1：确定评价标准",
  "message": "【环节1】确定评价标准\n\n任务：列出决策需要考虑的评价维度，并赋予权重\n时间盒：10分钟\n\n输出格式要求：\n## {你的Agent名称} - 建议的评价标准\n\n| 标准名称 | 权重(1-5) | 说明 |\n|----------|-----------|------|\n| {标准1} | {权重} | {说明} |\n\n请在{时间}内完成输出。"
}
```

### Participant输出模板

```markdown
## {Agent名称} - 建议的评价标准

| 标准名称 | 权重(1-5) | 说明 |
|----------|-----------|------|
| {标准1} | {权重} | {为什么重要} |
| {标准2} | {权重} | {为什么重要} |
| {标准3} | {权重} | {为什么重要} |
| {标准4} | {权重} | {为什么重要} |

### 标准选择理由
- {标准1}权重{N}的原因：{解释}
- {标准2}权重{N}的原因：{解释}
```

### TeamLead汇总模板

```markdown
## 决策矩阵 Phase 1 汇总

### 评价标准合并
| 标准 | 平均权重 | 提及次数 | 最终权重 |
|------|----------|----------|----------|
| {标准1} | {值} | {N} | {协商值} |
| {标准2} | {值} | {N} | {协商值} |

### 权重共识确认
- {DecisionMaker}确认权重分配：{标准权重列表}
- 权重总分：{总分}

### 权重分歧处理
- {标准}: {Agent A}建议{权重A}，{Agent B}建议{权重B}，取{协商值}
```

## Phase 3 - 方案评分

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "环节3：方案评分",
  "message": "【环节3】方案评分\n\n任务：对每个方案在各标准下评分(1-10分)，计算加权总分\n时间盒：15分钟\n\n输出格式要求：\n## {你的Agent名称} - 方案评分\n\n| 标准(权重) | 方案A | 方案B | 方案C |\n|-------------|-------|-------|-------|\n| {标准1}({权重}) | {分数} | {分数} | {分数} |\n\n| **加权总分** | **{总分}** | **{总分}** | **{总分}** |\n\n### 推荐方案\n- 推荐方案{X}，理由：{理由}\n\n请在{时间}内完成输出。"
}
```

### Participant输出模板

```markdown
## {Agent名称} - 方案评分

### 评分矩阵
| 标准(权重) | 方案A | 方案B | 方案C | 评分说明 |
|------------|-------|-------|-------|----------|
| {标准1}({权重}) | {分数} | {分数} | {分数} | {评分依据} |
| {标准2}({权重}) | {分数} | {分数} | {分数} | {评分依据} |
| {标准3}({权重}) | {分数} | {分数} | {分数} | {评分依据} |
| {标准4}({权重}) | {分数} | {分数} | {分数} | {评分依据} |
| **加权总分** | **{总分}** | **{总分}** | **{总分}** | |

### 推荐方案
- 推荐方案{X}
- 推荐理由：{基于分数的理由}
- 备选方案：{备选}
```

### TeamLead汇总模板

```markdown
## 决策矩阵 Phase 3 汇总

### 综合评分结果
| 方案 | 平均分 | 最高分Agent | 最低分Agent | 排名 |
|------|--------|-------------|-------------|------|
| A | {分} | {Agent}({分}) | {Agent}({分}) | {排名} |
| B | {分} | {Agent}({分}) | {Agent}({分}) | {排名} |
| C | {分} | {Agent}({分}) | {Agent}({分}) | {排名} |

### 评分差异分析
- {方案A}评分差异较大：{Agent X}{高分} vs {Agent Y}{低分}，原因：{分析}
- {方案B}评分较为一致：差异{值}

### 初步推荐
- 基于评分，推荐方案{X}
- 平均分最高：{方案}({分})
```

## Phase 4 - 综合决策

### TeamLead广播消息

```json
{
  "to": "decision-maker",
  "summary": "环节4：综合决策",
  "message": "【环节4】综合决策\n\n评分结果已汇总，请DecisionMaker确认最终决策。\n\n评分汇总：\n| 方案 | 平均分 | 排名 |\n|------|--------|------|\n| A | {分} | {排名} |\n| B | {分} | {排名} |\n\n请确认：\n1. 选择方案：{方案}\n2. 决策依据：{依据}\n3. 风险提示：{风险}"
}
```

### DecisionMaker输出模板

```markdown
## DecisionMaker - 最终决策确认

### 决策确认
- 选择方案：{方案}
- 决策依据：{基于评分和其他因素的综合考虑}
- 确认时间：{时间}

### 与评分差异说明
- {如选择非最高分方案}：选择{方案}而非最高分{方案}的原因：{说明}

### 执行条件
- 执行{方案}的前提条件：{条件}
- 风险预案：{预案}
```

```markdown
## 决策矩阵 Phase 4 汇总

- **选择方案**: {方案}
- **决策依据**: {依据}
- **DecisionMaker确认**: {确认人}于{时间}确认

### 决策记录
| 项目 | 内容 |
|------|------|
| 评价标准 | {标准列表及权重} |
| 候选方案 | {方案列表} |
| 最高分方案 | {方案}({分}) |
| 最终选择 | {方案} |
| 选择理由 | {理由} |

### 行动项
| Who | What | When |
|-----|------|------|
| {负责人} | {执行决策} | {时间} |
```

## 决策矩阵完整流程示例

### 环节推进时间盒建议

| 环节 | 时间盒 | 核心产出 |
|------|--------|----------|
| Phase 1 | 10分钟 | 评价标准及权重 |
| Phase 2 | 10分钟 | 候选方案列表 |
| Phase 3 | 15分钟 | 评分矩阵、推荐方案 |
| Phase 4 | 5分钟 | 最终决策确认 |
| **总计** | **40分钟** | **决策纪要** |

---

## 注意事项

- **权重主观性**：权重本身带主观性，需团队共识
- **评分一致性**：确保各Agent评分标准一致
- **分数是辅助**：分数只是辅助，最终决策权在Approver
- **差异讨论**：评分差异大时需讨论原因
