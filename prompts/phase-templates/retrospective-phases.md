# 敏捷回顾环节推进模板

## 使用说明

本模板用于敏捷回顾会(Retrospective)的环节推进。适用于迭代式团队的小步复盘。

---

## 回顾结构选择

| 变体 | 结构 |
|------|------|
| Stop/Start/Continue | 停止做 / 开始做 / 继续做 |
| 做得好/不好/改进 | 做得好的 / 做得不好的 / 尝试改进 |
| 4L法 | Liked / Learned / Lacked / Longed for |

---

## Phase 1 - 收集反馈

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "敏捷回顾：收集反馈",
  "message": "【敏捷回顾 Phase 1】收集反馈\n\n迭代/周期：{迭代名称}\n\n任务：按Stop/Start/Continue分类反馈\n时间盒：10分钟\n\n规则：具体、可操作，不空泛\n\n输出格式要求：\n## {你的Agent名称} - 敏捷回顾\n\n### Stop（停止）\n- {停止做的事}\n\n### Start（开始）\n- {新增做的事}\n\n### Continue（继续）\n- {保持做的事}\n\n### 最优先改进项\n- {优先级最高的改进}\n\n请在{时间}内完成输出。"
}
```

### Participant输出模板

```markdown
## {Agent名称} - 敏捷回顾

### Stop（停止）
- 停止项1: {具体描述}
- 停止项2: {具体描述}
- 停止项3: {具体描述}

### Start（开始）
- 开始项1: {具体描述}
- 开始项2: {具体描述}
- 开始项3: {具体描述}

### Continue（继续）
- 继续项1: {具体描述}
- 继续项2: {具体描述}

### 最优先改进项
- **最优先**: {改进项}
- **原因**: {为什么最优先}
```

### TeamLead汇总模板

```markdown

|------|----------|----------|
| {项目1} | {N} | {建议} |
| {项目2} | {N} | {建议} |

### Start高频项
| 项目 | 提及次数 | 具体建议 |
|------|----------|----------|
| {项目1} | {N} | {建议} |
| {项目2} | {N} | {建议} |

| 项目 | 提及次数 | 原因 |
|------|----------|------|
| {项目1} | {N} | {原因} |

| 改进项 | 提及次数 | 类型 |
```

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "敏捷回顾：汇总分析",

}
```

## Phase 3 - 行动项制定

### TeamLead广播消息

```json
{
  "to": "*",
  "summary": "敏捷回顾：行动项制定",
  "message": "【敏捷回顾 Phase 3】行动项制定\n\n任务：为P1改进项制定具体行动项（W3格式）\n时间盒：10分钟\n\nP1改进项：\n1. {改进1}\n2. {改进2}\n\n输出格式要求：\n## {你的Agent名称} - 行动项建议\n\n| Who | What | When | 改进类型 |\n|------|------|------|------|\n| {负责人} | {行动} | {时间} | {类型} |\n\n请在{时间}内完成输出。"
}
```

### Participant输出模板

```markdown
## {Agent名称} - 行动项建议

| Who | What | When | 改进类型 |
|-----|------|------|----------|
| {Agent} | {行动内容} | {截止日期} | Stop |
| {Agent} | {行动内容} | {截止日期} | Start |
| {Agent} | {行动内容} | {截止日期} | Start |

### 行动项说明
- {行动}: 对应改进项{改进}
- 执行方式：{方式}
```

```markdown

### 下迭代行动项（W3格式）

| Who | What | When | 类型 | 状态 |
|-----|------|------|------|------|
| {负责人} | {行动内容} | {截止日期} | Stop | 待执行 |
| {负责人} | {行动内容} | {截止日期} | Start | 待执行 |

### 回顾总结
| 项目 | 内容 |
|------|------|
| 迭代 | {迭代名称} |
| Stop项数 | {N} |
| Start项数 | {N} |
| Continue项数 | {N} |
| P1行动项数 | {N} |

### DecisionMaker确认
- 确认人：{DecisionMaker}
- 确认时间：{时间}
- 确认行动项：{确认内容}

### 下迭代验证点
- 在下迭代回顾时验证：
  - {行动项}是否完成
  - {改进}效果如何
```

## 敏捷回顾完整流程示例

### 环节推进时间盒建议

| 环节 | 时间盒 | 核心产出 |
|------|--------|----------|
| Phase 1 | 10分钟 | Stop/Start/Continue列表 |
| Phase 2 | 10分钟 | 改进优先级排序 |
| Phase 3 | 10分钟 | 行动项（W3格式） |
| **总计** | **30分钟** | **回顾纪要+行动项** |

---

## 注意事项

- **具体可操作**：不空泛，"改进沟通"不如"每日增加5分钟同步时间"
- **数量控制**：P1行动项不超过3个
- **下迭代验证**：每次回顾检查上迭代行动项
- **小步修正**：不追求宏大改进，小步快跑
- **无责原则**：不追责，只改进
