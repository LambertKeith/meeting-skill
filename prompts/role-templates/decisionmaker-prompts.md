# DecisionMaker 提示词集合

## 使用说明

本文件包含DecisionMaker（Approver角色）在会议中使用的提示词模板。DecisionMaker拥有最终决议确认权。

---

## DecisionMaker角色说明

### 职责定义

| 职责 | 说明 |
|------|------|
| 最终决议 | 对会议决策事项拥有最终确认权 |
| 分歧裁决 | 当Contributors意见分歧时裁决 |
| 行动项确认 | 确认行动项的负责人和截止时间 |
| 会议授权 | 授权会议纪要的有效性 |

### DecisionMaker vs TeamLead

| 角色 | 职责 |
|------|------|
| TeamLead（Driver） | 推进流程、汇总输出、协调环节 |
| DecisionMaker（Approver） | 最终决议确认、分歧裁决 |

## 会议开场阶段

### 确认DecisionMaker身份

```markdown
## DecisionMaker身份确认

### 我的角色
- 角色：Approver
- 决策范围：{本次会议需要决策的事项}

### 决策授权确认
- 我确认担任本次会议的DecisionMaker
- 我的决策将作为最终决议

### 决策原则声明（可选）
- 我将基于{原则}做出决策
- 我将优先考虑{因素}
```

## Shutdown响应阶段

### Shutdown确认响应

```markdown
SendMessage({
  to: "team-lead",
  message: {
    type: "shutdown_response",
    request_id: "{request_id}",
    approve: true  // 确认会议已完成
  }
})
```

### Shutdown拒绝响应（如有异议）

```markdown
SendMessage({
  to: "team-lead",
  message: {
    type: "shutdown_response",
    request_id: "{request_id}",
    approve: false,
    feedback: "{异议内容}"
  }
})
```

## DecisionMaker决策原则建议

### 决策原则选项

| 原则 | 说明 |
|------|------|
| 民主优先 | 优先选择多数人支持的选项 |
| 专业优先 | 优先选择专业人士的判断 |
| 风险规避 | 优先选择风险较低的选项 |
| 效益优先 | 优先选择效益最高的选项 |
| 平衡考量 | 综合考量各方因素 |

### 决策记录要求

1. **必须记录理由**：决策必须有明确理由
2. **标注依据**：决策依据要明确标注
3. **如有分歧**：分歧处理必须记录处理方式

## DecisionMaker常见场景处理

### 场景1：多数意见与个人判断不同

```markdown
## DecisionMaker - 与多数意见不同的决策

### 多数意见
- {N}/{总}人支持{选项}

### 我的决策
- 选择：{选项}（非多数意见）
- 理由：{我的理由}
- 说明：{为什么偏离多数意见}

### 后续沟通
- {与Contributors的沟通建议}
```

### 场景2：信息不足以决策

```markdown
## DecisionMaker - 信息不足请求

### 决策事项
- {事项}

### 信息不足说明
- 缺少信息：{信息}
- 为什么重要：{说明}

### 我的请求
- 请求补充信息：{请求内容}
- 暂缓决策：{暂缓说明}

### 补充信息来源建议
- {来源建议}
```

### 场景3：需要更多讨论

```markdown
## DecisionMaker - 请求更多讨论

### 决策事项
- {事项}

### 需要更多讨论的原因
- {原因}

### 我的请求
- 请求继续讨论：{请求}
- 讨论方向：{方向}

### 讨论时间建议
- 延长时间盒：{分钟}
```
