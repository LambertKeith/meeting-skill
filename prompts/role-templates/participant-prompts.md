# Participant 提示词集合

## 使用说明

本文件包含Participant（Contributor角色）在会议各阶段使用的提示词和输出模板。

---

## 会议开场阶段

### 接收开场广播后的响应

```markdown
## {我的Agent名称} - 会议确认

### 准备就绪
- 已阅读议程
- 已理解角色
- 已确认会议目标

### 我的角色认知
- 角色：Contributor
- 职责：提供专业意见、参与环节讨论

### 补充信息（如有）
- {我对议题的补充背景信息}

### 期望说明（如有）
- {我对会议的期望}
```

---

## 环节响应阶段

### 环节输出基本原则

1. **按时输出**：在时间盒内完成输出
2. **格式正确**：严格按照要求的格式输出
3. **内容充实**：提供有价值的专业意见
4. **引用他人**：可延展他人观点，标注来源

### 环节输出通用模板

```markdown
## {我的Agent名称} - {环节名称}

### {主要内容标题}
1. {内容项1}: {描述}
2. {内容项2}: {描述}
3. {内容项3}: {描述}

### {补充内容标题}
- {补充项}: {描述}

### 延展他人观点（如有）
- 基于{Agent X}的观点{编号}：延展为{内容}
```

## 延展他人观点规范

### 正确延展格式

```markdown
### 延展他人观点
- 基于{Agent名称}的观点{编号}：延展为{我的延展内容}
- 原观点：{原观点摘要}
- 延展内容：{我的延展}
- 延展理由：{为什么延展}
```

### 延展原则

1. **标注来源**：明确标注延展的原观点来源
2. **有价值延展**：延展要有实际增量，不是简单重复
3. **尊重原观点**：不歪曲原观点含义

## Shutdown响应规范

### Shutdown响应格式

```markdown
SendMessage({
  to: "team-lead",
  message: {
    type: "shutdown_response",
    request_id: "{request_id}",
    approve: true  // 或 false，如有未完成工作则false
  }
})
```

### Shutdown检查点

在响应Shutdown前检查：
1. 所有环节输出是否已提交
2. 是否有待补充的内容
3. 是否同意会议纪要

如有异议：
```markdown
{
  type: "shutdown_response",
  request_id: "{request_id}",
  approve: false,
  feedback: "纪要中{某点}未准确反映我的观点，建议修正为{修正内容}"
}
```
