# 会议议程模板

## 使用说明

本模板用于会议议程文档的标准化输出。TeamLead在会议初始化阶段使用此模板创建agenda.md。

---

## 模板格式

```markdown
# 会议议程

## 基本信息

| 项目 | 内容 |
|------|------|
| 会议类型 | {类型：问题分析会/决策会/方案评审会/创意脑暴会/复盘会/共识对齐会/日常站会} |
| 会议议题 | {议题描述} |
| 会议目标 | {目标描述} |
| 预期产出 | {产出描述} |
| 会议方法 | {方法组合} |
| 时间估计 | {总时长估计} |

## 会议背景

### 问题/议题描述
{详细描述}

### 相关材料
- {材料1}: {链接或描述}
- {材料2}: {链接或描述}

### 历史决议引用（如有）
- 上次会议决议：{内容}
- 待决事项：{内容}

## 议程环节

| 序号 | 环节名称 | 方法 | 时间盒 | 预期产出 | 模板引用 |
|------|----------|------|--------|----------|----------|
| 1 | {环节} | {方法} | {分钟}min | {产出} | {phase-template路径} |
| 2 | {环节} | {方法} | {分钟}min | {产出} | {phase-template路径} |
| 3 | {环节} | {方法} | {分钟}min | {产出} | {phase-template路径} |
| ... | ... | ... | ... | ... | ... |

## 议程说明

### 环节1：{环节名称}
- **目标**: {环节目标}
- **方法说明**: {方法简要说明}
- **产出要求**: {产出格式}

### 环节2：{环节名称}
- **目标**: {环节目标}
- **方法说明**: {方法简要说明}
- **产出要求**: {产出格式}

## 会议角色（见roles.md）

| 角色 | Agent | 职责 |
|------|-------|------|
| Driver | TeamLead | 推进流程 |
| Approver | {DecisionMaker} | 最终决议 |
| Contributor | {Participants} | 提供意见 |

## 会议规则

### 时间盒规则
- 各环节严格按时间盒执行
- 超时需申请延时
- TeamLead控制节奏

### 发言规则
- 按环节顺序发言
- 不打断他人发言
- 可延展他人观点

### 决策规则
- DecisionMaker拥有最终决议权
- 分歧由DecisionMaker裁决
- 决议需记录理由

## 文件产出清单

会议结束后将产出：
1. agenda.md - 本议程文档
2. roles.md - 角色分配文档
3. phases/ - 各环节输出文档
4. process-log.md - 流程日志
5. minutes.md - 会议纪要
6. action-items.md - 行动项清单

---
议程创建时间：{时间}
议程创建者：TeamLead
```

## 各会议类型议程示例

### 问题分析会议程

```markdown
# 会议议程 - 问题分析会

## 基本信息
| 项目 | 内容 |
|------|------|
| 会议类型 | 问题分析会 |
| 会议议题 | {问题描述} |
| 会议目标 | 分析问题根因，制定解决方案 |
| 预期产出 | 根因分析报告、行动项清单 |
| 会议方法 | ORID + 5Why + 鱼骨图 |
| 时间估计 | 40分钟 |

## 议程环节
| 序号 | 环节 | 方法 | 时间盒 | 预期产出 | 模板引用 |
|------|------|------|--------|----------|----------|
| 1 | 客观事实收集 | ORID-O | 10min | 事实列表 | prompts/phase-templates/orid-phases.md |
| 2 | 感受反应 | ORID-R | 5min | 情绪分布 | prompts/phase-templates/orid-phases.md |
| 3 | 根因分析 | ORID-I + 5Why | 15min | 根因识别 | prompts/phase-templates/orid-phases.md |
| 4 | 决策行动 | ORID-D | 10min | 行动项 | prompts/phase-templates/orid-phases.md |
```

## 议程创建检查点

创建议程后检查：

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 基本信息完整 | ✅/❌ | 类型/议题/目标/产出 |
| 议程环节时间盒合理 | ✅/❌ | 总时长是否合理 |
| 方法与环节匹配 | ✅/❌ | 方法是否适合环节目标 |
| 模板引用正确 | ✅/❌ | phase-template路径是否正确 |
| DecisionMaker已指定 | ✅/❌ | 是否明确Approver |
