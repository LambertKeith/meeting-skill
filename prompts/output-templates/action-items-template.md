# 行动项模板（W3格式）

## 使用说明

本模板用于行动项文档的标准化输出。TeamLead在会议收尾阶段使用此模板创建action-items.md。采用W3格式（Who/What/When）。

---

## W3格式说明

| 元素 | 说明 | 要求 |
|------|------|------|
| Who | 执行负责人 | 必须明确指定Agent或人员 |
| What | 行动内容 | 具体可操作，不空泛 |
| When | 截止时间 | 明确日期或相对时间 |

---

## 行动项追踪机制

### 状态定义

| 状态 | 说明 | 触发条件 |
|------|------|----------|
| 待执行 | 新创建 | 会议结束 |
| 进行中 | 开始执行 | 负责人报告 |
| 已完成 | 完成执行 | 负责人确认 |
| 已取消 | 不再执行 | 新会议决议 |
| 已延期 | 超期未完成 | 截止日期后 |
| 已阻塞 | 遇到障碍 | 负责人报告 |

### 追踪规则

1. **负责人主动更新**: 负责人需主动更新行动项状态
2. **状态更新频率**: 每周至少更新一次状态
3. **超期说明**: 超期需说明原因和新的预计完成时间
4. **阻塞报告**: 遇到阻塞需立即报告

### 下次会议检查

下次会议将检查：
- 上次会议行动项完成状态
- 延期或阻塞的行动项原因
- 新的行动项状态确认

## 行动项创建检查点

创建行动项后检查：

| 检查项 | 状态 | 说明 |
|--------|------|------|
| 每项有明确负责人 | ✅/❌ | Who已指定 |
| 每项具体可操作 | ✅/❌ | What不空泛 |
| 每项有截止时间 | ✅/❌ | When已指定 |
| 来源已标注 | ✅/❌ | 来自哪个环节/决议 |
| 优先级已分级 | ✅/❌ | P1/P2/P3已区分 |
| 依赖关系已识别 | ✅/❌ | 依赖项已标注 |
| DecisionMaker已确认 | ✅/❌ | 分配合理性已确认 |
