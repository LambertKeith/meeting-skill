# 罗伯特议事规则

## 方法概述

最经典的正式议事体系，核心目的是保证程序公平、讨论有序、结论合法有效。

## 基本流程

| 步骤 | 动作 | 说明 |
|------|------|------|
| 1 | 提出动议 | 明确要讨论的议题 |
| 2 | 附议 | 他人确认议题值得讨论 |
| 3 | 辩论 | 各方表达意见 |
| 4 | 修正动议 | 必要时修改议题内容 |
| 5 | 表决 | 形成正式决议 |

## Agent适配流程

### Phase 1 - 动议提出

**输出模板**：
```markdown
## 动议提案

### 动议内容
{明确的议题表述}

### 动议理由
{为什么要讨论这个议题}

### 预期产出
{希望达成什么决议}
```

### Phase 2 - 附议确认

**输出模板**：
```markdown
## 附议确认

### 是否附议
- [附议/不附议]

### 附议理由
{简要说明为何值得进入讨论}
```

### Phase 3 - 辩论环节

**规则**：
- 每人发言不超过{次数}次
- 每次发言不超过{时间}
- 发言必须针对动议本身

**输出模板**：
```markdown
## {Agent名称} - 辩论发言

### 立场
- [支持/反对/中立]

### 论点
1. ...
2. ...

### 建议修正
{如有修正建议，在此提出}
```

### Phase 4 - 动议修正

**输出模板**：
```markdown
## 修正动议

### 原动议
{原文}

### 修正内容
{修正后的动议}

### 修正理由
{为何需要修正}
```

### Phase 5 - 表决

**输出模板**：
```markdown
## 表决结果

| Agent | 投票 | 理由 |
|-------|------|------|
| A | 支持/反对 | ... |
| B | 支持/反对 | ... |

### 决议
- 结果: {通过/否决}
- 决议内容: {最终通过的动议文本}
- 决策者确认: {Approver签名}
```

## 适用场景

- 正式决策会议
- 委员会/董事会
- 权责敏感议题
- 需要结果可追溯的场合

## 简化使用

对非正式场合，可简化为：
1. 提出议题
2. 确认是否进入讨论
3. 限制发言轮次
4. 归纳修正
5. 明确决策权
6. 形成决议
