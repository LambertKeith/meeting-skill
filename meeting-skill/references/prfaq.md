# PRFAQ 逆向写作法

## 方法概述

假设产品已成功发布，反过来写新闻稿(PR)和常见问题(FAQ)。迫使团队站在用户价值和落地场景思考，而非陷在内部功能争论。

## 核心价值

| 部分 | 内容 | 作用 |
|------|------|------|
| PR | 新闻稿 | 从用户视角描述价值 |
| FAQ | 常见问题 | 回答关键疑问 |

## Agent适配流程

### Phase 1 - 写新闻稿

**TeamLead广播**：
```
【PRFAQ】产品/方案：{名称}
任务：假设已成功发布，写新闻稿
视角：站在用户和媒体视角
规则：不写内部功能，写用户价值
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 新闻稿 (PR)

### 标题
{吸引人的新闻标题}

### 导语
{一句话描述核心价值}

### 正文
{新闻稿正文：用户场景、价值、影响}

### 用户引言
{虚构的用户好评}

### 发布信息
{发布时间、渠道等}

### 核心价值总结
{用户得到什么}
```

### Phase 2 - 写FAQ

**TeamLead广播**：
```
【PRFAQ】写FAQ
任务：回答关键疑问
涵盖：用户、媒体、内部疑问
```

**Contributor输出模板**：
```markdown
## {Agent名称} - FAQ

### 用户问题
| 问题 | 回答 |
|------|------|
| 这是什么？ | ... |
| 对我有什么好处？ | ... |
| 怎么使用？ | ... |
| 费用多少？ | ... |

### 媒体/分析问题
| 问题 | 回答 |
|------|------|
| 为什么做这个？ | ... |
| 与竞品差异？ | ... |

### 内部/执行问题
| 问题 | 回答 |
|------|------|
| 技术可行性？ | ... |
| 需要什么资源？ | ... |
```

### Phase 3 - 汇总讨论

**TeamLead汇总**：
```markdown
## PRFAQ汇总

### 最佳PR草稿
{综合各Agent的PR}

### 关键FAQ
{综合各Agent的FAQ}

### 讨论结论
- 核心用户价值：{价值}
- 需回答的关键问题：{问题}
- 实现需解决的问题：{问题}

### 行动项
| 序号 | Who | What | When |
|------|-----|------|------|
| ... | ... | ... | ... |
```

## 适用场景

- 产品创新讨论
- 新功能设计
- 业务方向讨论
- 避免陷入内部功能争论

## 注意事项

- PR必须站在用户视角，不写内部功能
- FAQ必须覆盖真实疑问
- 先写PR再讨论，防止陷入细节争论
- 帮助团队理解用户价值
- 逆向思考，从结果推过程
