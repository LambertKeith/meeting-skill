# 设计冲刺 (Design Sprint)

## 方法概述

完整创新解决流程，快速理解问题、生成方案、选定方向、做出原型、获取测试反馈。

## 五阶段结构（可压缩）

| 阶段 | 内容 | Agent任务 |
|------|------|-----------|
| 1 | 理解问题 | 统一问题地图 |
| 2 | 发散构思 | 独立构思方案 |
| 3 | 决定方向 | 投票筛选方案 |
| 4 | 制作原型 | 产出原型方案 |
| 5 | 测试验证 | 获取反馈改进 |

## Agent适配流程

### Day 1 - 理解问题

**TeamLead广播**：
```
【设计冲刺 Day 1】理解问题
任务：
1. 梳理问题背景
2. 定义成功标准
3. 绘制问题地图
输出：问题地图文档
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 问题理解

### 问题背景
{背景描述}

### 成功标准
- {标准1}
- {标准2}

### 问题地图
- 核心问题: ...
- 子问题:
  - ...
  - ...

### 关键假设
- {假设1}
- {假设2}
```

### Day 2 - 发散构思

**TeamLead广播**：
```
【设计冲刺 Day 2】发散构思
任务：独立构思解决方案
规则：不讨论，各自独立产出
时间：{分钟}
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 方案构思

### 方案A
- 核心思路: ...
- 关键步骤: ...
- 预期效果: ...

### 方案B
- ...

### 方案C
- ...
```

### Day 3 - 决定方向

使用点投票或决策矩阵筛选。

**TeamLead汇总**：
```markdown
## 方案筛选结果

### 进入原型阶段：{方案X}
- 提出者: {Agent}
- 理由: {投票结果}
```

### Day 4 - 制作原型

**TeamLead广播**：
```
【设计冲刺 Day 4】制作原型
任务：将选定方案细化为可执行原型
输出：完整方案文档（可执行）
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 原型方案

### 方案：{方案名称}

### 详细设计
{完整可执行的设计}

### 执行步骤
| 步骤 | 内容 | 输出 |
|------|------|------|
| 1 | ... | ... |

### 验证要点
- {要验证的关键假设}
```

### Day 5 - 测试验证

**TeamLead广播**：
```
【设计冲刺 Day 5】测试验证
任务：设计验证方法，获取反馈
输出：验证报告和改进建议
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 验证建议

### 验证方法
- {如何验证}

### 验证指标
| 指标 | 期望值 | 验证方式 |
|------|--------|----------|
| ... | ... | ... |

### 改进建议
- ...
```

## 适用场景

- 创新产品设计
- 新功能开发
- 业务突破项目
- 需要快速验证的场景

## 注意事项

- 不停留在想法层，要求走向可验证方案
- 可压缩为更短时间（如半天一轮）
- 核心逻辑：问题→发散→收敛→原型→验证
