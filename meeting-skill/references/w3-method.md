# W3 方法 (Who / What / When)

## 方法概述

极简执行框架，每个议题结束后必须落到三件事：谁做、做什么、什么时候完成。

## 三要素

| 要素 | 内容 | 要求 |
|------|------|------|
| Who | 责任人 | 明确具体的人 |
| What | 任务内容 | 具体可执行的动作 |
| When | 完成时间 | 明确的时间点 |

## Agent适配流程

### 形成行动项

**TeamLead广播**：
```
【W3】形成行动项
任务：将讨论结论转化为具体行动项
规则：每项必须有明确的Who/What/When
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 行动项建议

| 序号 | Who | What | When |
|------|-----|------|------|
| 1 | {责任人} | {任务} | {时间} |
| 2 | {责任人} | {任务} | {时间} |
```

### 行动项汇总

**TeamLead汇总确认**：
```markdown
## 行动项清单 (W3)

| 序号 | Who | What | When | 状态 | 备注 |
|------|-----|------|------|------|------|
| 1 | Agent A | ... | 2026-04-10 | 待执行 | |
| 2 | Agent B | ... | 2026-04-12 | 待执行 | |
| 3 | Agent C | ... | 2026-04-15 | 待执行 | |

### Approver确认
- 行动项认可: ✅
- 签名: {Approver}
```

### 后续跟踪

```markdown

| 序号 | Who | What | When | 状态 | 完成时间 |
|------|-----|------|------|------|----------|
| 1 | Agent A | ... | 2026-04-10 | 完成 | 2026-04-09 |

| 3 | Agent C | ... | 2026-04-15 | 待执行 | - |

### 阻塞项
- 行动项{N}: {阻塞原因}
```

## 适用场景

- 几乎所有会议的最后环节
- 项目推进会
- 确保讨论有产出
- 防止会议"热闹但无结果"

## 注意事项

- Who必须是具体的人/Agent
- What必须是可执行的动作
- When必须是明确的时间点
- 不写"尽快"、"尽早"等模糊时间
- 每次会议必须有行动项输出
