# 时间盒管理 (Timeboxing)

## 方法概述

给每个议题设置严格时间上限，时间到必须进入收束、决策或挂起。

## 核心价值

- 逼迫团队聚焦关键信息
- 压缩无效表达
- 防止次要议题占用过多时间

## 常见时间盒配置

| 议题类型 | 典型配置 |
|----------|----------|
| 简议题 | 5min背景 + 10min讨论 + 5min结论 |
| 中议题 | 10min背景 + 15min质询 + 15min讨论 + 5min结论 |
| 复议题 | 15min背景 + 20min讨论 + 15min决策 |
| 方案陈述 | 每个≤3min |
| 评论发言 | 每个≤1min |

## Agent适配流程

### 设定时间盒

在议程中明确每个议题的时间分配：
```markdown
| 序号 | 议题 | 时间盒 | 各阶段时间 |
|------|------|--------|------------|
| 1 | ... | 20min | 5+10+5 |
| 2 | ... | 45min | 10+15+15+5 |
```

### 执行时间盒

**TeamLead推进**：
```
【议题{N}】时间盒开始
总时间: {分钟}
分配:
- 输入阶段: {分钟}
- 讨论阶段: {分钟}
- 结论阶段: {分钟}
```

**时间提醒**：
```
【时间提醒】议题{N}
剩余时间: {分钟}
当前阶段: {阶段}
请尽快完成{阶段任务}。
```

**时间盒结束**：
```
【时间盒结束】议题{N}
当前状态: {讨论进展}
处理方式:
- [已达成结论 / 挂起待议 / 延长时间(需Approver批准)]
```

### 挂起处理

```markdown
## 挂起议题记录

| 议题 | 挂起原因 | 待议时间 | 优先级 |
|------|----------|----------|--------|
| ... | {未解决原因} | {下次会议时间} | P1/P2/P3 |
```

## 适用场景

- 所有会议的时间控制
- 防止发言过长
- 多议题会议的节奏控制

## 注意事项

- Driver必须严格执行，不能无限延长
- 延长时间需Approver批准
- 挂起议题要记录并安排后续处理
- 时间紧迫时，优先处理核心议题
