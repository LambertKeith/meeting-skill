# ORID 焦点讨论法

## 方法概述

ORID将讨论分为四层，防止一上来就争论观点，强制先讲清事实。

| 层级 | 名称 | 内容 | Agent输出 |
|------|------|------|-----------|
| O | Objective | 客观事实 | 列出发生了什么、数据、现象 |
| R | Reflective | 感受反应 | 对事实的感受、情绪、联想 |
| I | Interpretive | 理解判断 | 分析意义、原因、关联 |
| D | Decisional | 决策行动 | 结论、下一步、行动项 |

## Agent适配流程

### Phase O - 客观事实

**TeamLead广播**：
```
【环节O】客观事实收集
任务：列出所有客观事实、数据、现象，不掺杂观点
输出格式：
- 事实1: ...
- 事实2: ...
- 数据: ...
```

**Participant输出模板**：
```markdown
## {Agent名称} - 客观事实

### 观察到的事实
1. ...
2. ...
3. ...

### 相关数据
- ...

### 事件时间线
- ...
```

### Phase R - 感受反应

**输出模板**：
```markdown
## {Agent名称} - 感受反应

### 对关键事实的反应
- 事实X令我...因为...
- ...

### 惊讶/担忧/期待
- ...
```

### Phase I - 理解判断

**输出模板**：
```markdown
## {Agent名称} - 理解判断

### 核心洞察
1. ...
2. ...

### 原因分析
- ...

### 意义解读
- ...
```

### Phase D - 决策行动

**输出模板**：
```markdown
## {Agent名称} - 决策行动

### 结论
- ...

### 建议行动
| 序号 | 行动 | 责责人 | 时间 |
|------|------|--------|------|
| 1 | ... | ... | ... |

### 待决问题
- ...
```

## 适用场景

- 项目复盘
- 问题分析
- 共识建立
- 事件总结

## 注意事项

- O阶段禁止表达观点，只列事实
- R阶段控制篇幅，避免情绪宣泄
- I阶段要有依据，不能空谈
- D阶段必须产出可执行项
