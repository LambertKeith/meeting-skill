# 敏捷回顾会 (Retrospective)

## 方法概述

迭代式团队的复盘方法，强调小步修正而非宏大总结。

## 常见结构

### Stop / Start / Continue

| 类别 | 内容 | Agent任务 |
|------|------|-----------|
| Stop | 哪些要停止 | 停止做的事 |
| Start | 哪些要开始 | 新增要做的事 |
| Continue | 哪些要继续 | 保持做的事 |

### 其他变体

| 变体 | 内容 |
|------|------|
| 做得好 | 本次做得好的 |
| 做得不好 | 本次做得不好的 |
| 尝试改进 | 接下来尝试的 |

## Agent适配流程

### Phase 1 - 收集反馈

**TeamLead广播**：
```
【敏捷回顾】{迭代/周期}
任务：按Stop/Start/Continue分类反馈
规则：具体、可操作，不空泛
时间：{分钟}
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 敏捷回顾

### Stop（停止）
- {停止做的事}
- ...

### Start（开始）
- {新增做的事}
- ...

### Continue（继续）
- {保持做的事}
- ...

### 最优先改进项
- {优先级最高的改进}
```

### Phase 2 - 汇总分析

**TeamLead汇总**：
```markdown
## 回顾汇总

### Stop高频项
| 项目 | 提及次数 | 建议 |
|------|----------|------|
| ... | {N} | ... |

### Start高频项
| 项目 | 提及次数 | 建议 |
|------|----------|------|
| ... | {N} | ... |

### Continue高频项
| 项目 | 提及次数 | 原因 |
|------|----------|------|
| ... | {N} | ... |

### 改进优先级排序
| 序号 | 改进 | 类型 | 优先级 |
|------|------|------|--------|
| 1 | ... | Stop | P1 |
| 2 | ... | Start | P1 |
```

### Phase 3 - 行动项

**TeamLead输出**：
```markdown
## 下迭代行动项

| 序号 | Who | What | When | 类型 |
|------|-----|------|------|------|
| 1 | ... | ... | ... | Stop |
| 2 | ... | ... | ... | Start |
```

## 适用场景

- 迭代式团队
- 每次迭代结束
- 定期复盘
- 小步改进

## 注意事项

- 具体可操作，不空泛
- 数量控制，不太多
- 每次只选少数最优先改进
- 下迭代验证改进效果
- 强调小步修正，不追求宏大
