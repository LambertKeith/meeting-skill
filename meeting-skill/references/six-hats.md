# 六顶思考帽

## 方法概述

用不同"帽子"代表不同思考角度，让团队在同一时间只做一种类型的思考。

| 帽子 | 颜色 | 思考类型 | Agent任务 |
|------|------|----------|-----------|
| 白帽 | 白色 | 事实数据 | 列出已知事实和数据 |
| 红帽 | 红色 | 感受直觉 | 表达直觉感受，不需解释 |
| 黑帽 | 黑色 | 风险批判 | 指出风险、问题、隐患 |
| 黄帽 | 黄色 | 价值机会 | 指出价值、收益、机会 |
| 绿帽 | 绿色 | 创意方案 | 提出新想法、新方案 |
| 蓝帽 | 蓝色 | 过程控制 | 总结、规划下一步 |

## Agent适配流程

**TeamLead按帽子顺序推进**：

```
【帽子{颜色}】{思考类型}
时间盒：{分钟}
任务：{具体任务}
规则：{该帽子的规则}
请各Participant输出。
```

### 白帽环节

```
任务：列出所有已知事实、数据、信息
规则：
- 只陈述事实，不表达观点
- 区分"已知"和"需要知道"
- 使用数据支撑
```

### 红帽环节

```
任务：表达对议题的直觉感受
规则：
- 允许表达感受，不需解释原因
- 时间盒严格控制（通常5分钟）
- 不争论，不质疑
```

### 黑帽环节

```
任务：指出风险、问题、隐患
规则：
- 关注逻辑上的缺陷
- 指出可能失败的原因
- 评估可行性风险
```

### 黄帽环节

```
任务：指出价值、收益、机会
规则：
- 寻找积极面
- 评估潜在收益
- 不必过度乐观，但要发现价值
```

### 绿帽环节

```
任务：提出新想法、新方案
规则：
- 不评价他人想法
- 数量优先
- 可以在他人想法上延伸
```

### 蓝帽环节

```
任务：总结、形成结论、规划下一步
规则：
- 归纳各帽子产出
- 形成结论
- 明确下一步
```

## 适用场景

- 复杂议题讨论
- 方案评审
- 容易争执的场景
- 多角色团队决策

## 推荐顺序

**问题分析**: 白 → 红 → 黑 → 黄 → 绿 → 蓝
**方案评估**: 白 → 黄 → 黑 → 绿 → 蓝
**创意发散**: 绿 → 黄 → 黑 → 蓝
