# 头脑风暴 (Brainstorming)

## 方法概述

最常见的创意发散方法，目标是提高方案数量、质量和多样性。

## 经典规则

| 规则 | 说明 |
|------|------|
| 延迟评价 | 不立刻评判好坏 |
| 鼓励数量 | 先追求想法数量 |
| 欢迎异想天开 | 奇怪想法也欢迎 |
| 延展他人想法 | 在别人想法上继续发展 |

## Agent适配流程

### Phase 1 - 明确问题

**TeamLead广播**：
```
【头脑风暴】{问题/目标}
任务：提出尽可能多的想法
规则：
- 不评价他人想法
- 数量优先
- 可在他人想法上延展
时间盒：{分钟}
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 头脑风暴

### 想法列表（不评价，只列出）

1. {想法1}
2. {想法2}
3. {想法3}
...
N. {想法N}

### 延展他人想法
- 基于{Agent X}的想法{N}：{延展内容}
```

### Phase 2 - 汇总展示

**TeamLead汇总**：
```markdown
## 头脑风暴汇总

### 想法总数：{N}

### 想法分类
| 类别 | 想法数 | 代表想法 |
|------|--------|----------|
| ... | ... | ... |

### 所有想法列表
| 序号 | 想法 | 提出者 | 被延展次数 |
|------|------|--------|------------|
| 1 | ... | Agent A | {N} |
| 2 | ... | Agent B | {N} |
```

### Phase 3 - 筛选收敛

使用点投票或其他方法筛选。

## 适用场景

- 创新设计
- 产品构思
- 业务突破
- 命名、营销活动策划

## 注意事项

- 需清晰问题定义
- 需时间限制
- 需后续筛选机制
- 防止变成"轮流随口说"
- 防止领导一开口别人不敢偏离
