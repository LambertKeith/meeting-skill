# SCAMPER 法

## 方法概述

实用创意检查框架，通过七类动作刺激改造思路。适合已有产品/流程的改进。

## 七类动作

| 动作 | 英文 | 问题 |
|------|------|------|
| 替代 | Substitute | 能替代什么？用什么替代？ |
| 组合 | Combine | 能和什么组合？组合后效果？ |
| 调整 | Adapt | 能借用什么？如何调整适用？ |
| 修改 | Modify | 能放大/缩小/改变什么？ |
| 另用 | Put to another use | 能另作什么用途？ |
| 删除 | Eliminate | 能删掉什么？简化什么？ |
| 反转 | Reverse/Rearrange | 能反转什么？重组什么？ |

## Agent适配流程

### Phase 1 - 确定改进对象

**TeamLead广播**：
```
【SCAMPER】改进对象：{产品/流程}
任务：按七类动作逐一思考改进可能性
```

### Phase 2 - 七类思考

**Contributor输出模板**：
```markdown
## {Agent名称} - SCAMPER分析

### 改进对象：{对象}

| 动作 | 思考 | 改进建议 |
|------|------|----------|
| 替代 | {能替代什么} | {建议} |
| 组合 | {能组合什么} | {建议} |
| 调整 | {能借用什么} | {建议} |
| 修改 | {能修改什么} | {建议} |
| 另用 | {能另作何用} | {建议} |
| 删除 | {能删掉什么} | {建议} |
| 反转 | {能反转什么} | {建议} |

### 最有价值的改进
- 动作类型: {类型}
- 改进内容: {内容}
- 预期效果: {效果}
```

### Phase 3 - 汇总筛选

**TeamLead汇总**：
```markdown
## SCAMPER汇总

### 各动作高频建议
| 动作 | 高频建议 | 提及次数 |
|------|----------|----------|
| 删除 | ... | {N} |
| 组合 | ... | {N} |

### 进入试行的改进
| 序号 | 改进 | 动作类型 | 来源 |
|------|------|----------|------|
| 1 | ... | 删除 | Agent A, B |
```

## 适用场景

- 已有产品改进
- 已有流程优化
- 服务升级
- 不适合完全从零发明

## 注意事项

- 逐一思考，不跳过
- 每类动作都可能有价值
- 删除和简化往往最有价值
