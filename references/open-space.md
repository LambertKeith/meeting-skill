# 开放空间会议 (Open Space Technology)

## 方法概述

参与者自己提出议题，按兴趣加入不同讨论组，最后回收成果。最大化自组织活力。

## 四原则

| 原则 | 说明 |
|------|------|
| 参与者是谁，就是对的 | 任何人都可以参与 |
| 发生什么，就是该发生的 | 不预设结果 |
| 何时开始，就是对的时间 | 自然开始 |
| 结束时，就结束了 | 自然结束 |

## 一法则

**双脚法则**：如果不贡献或学习，用双脚走到更有价值的地方。

## Agent适配流程

### Phase 1 - 议题提议

**TeamLead广播**：
```
【开放空间】议题提议
任务：每人提出感兴趣的议题
规则：不限制，自由提议
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 议题提议

### 议题1
- 主题: ...
- 简介: ...
- 我为什么关心: ...

### 议题2
- ...
```

### Phase 2 - 议题公告

**TeamLead汇总议题，形成议题墙**：
```markdown
## 议题墙

| 议题ID | 议题 | 提议者 | 关注者 |
|--------|------|--------|--------|
| A | ... | Agent X | Agent Y, Z |
| B | ... | Agent M | Agent N |
| C | ... | Agent P | ... |
```

### Phase 3 - 自组讨论

**TeamLead广播**：
```
【开放空间】自由组队讨论
规则：按兴趣加入议题组
时间：{分钟}
各组自行记录讨论成果
```

**小组输出模板**：
```markdown
## 议题{ID}讨论记录

### 参与者
- {Agent列表}

### 讨论内容
{讨论过程}

### 成果/结论
- ...

### 待后续事项
- ...
```

### Phase 4 - 成果回收

**TeamLead汇总**：
```markdown
## 开放空间成果汇总

### 各议题成果
| 议题 | 参与人数 | 主要成果 | 后续建议 |
|------|----------|----------|----------|
| A | {N} | ... | ... |
| B | {N} | ... | ... |

### 高关注议题
- ...

### 待跟进事项
- ...
```

## 适用场景

- 大型内部研讨
- 社区型组织
- 创新文化强的团队
- 议题开放、参与者专业性强

## 注意事项

- 不适合执行型、时限紧的会议
- 需要强自组织能力
- 最后需回收成果并跟进
- 议题质量取决于参与者
