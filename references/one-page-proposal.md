# 一页纸提案法

## 方法概述

节奏较快但仍希望有结构输入的方法，把提案压缩在一页内。

## 一页纸结构

| 部分 | 内容 | 占比 |
|------|------|------|
| 背景 | 为什么提这个提案 | 20% |
| 目标 | 要达成什么 | 10% |
| 问题 | 当前问题是什么 | 20% |
| 方案 | 具体方案 | 30% |
| 资源 | 需要什么资源 | 10% |
| 风险 | 可能风险 | 5% |
| 结论 | 建议 | 5% |

## Agent适配流程

### 会前准备

**TeamLead广播**：
```
【一页纸提案】议题：{议题}
任务：撰写一页纸提案
规则：简洁、完整、结构化
提交时间：{时间}
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 一页纸提案

### 背景
{为什么提这个提案，一两句话}

### 目标
{要达成什么，一两句话}

### 问题
{当前问题是什么，简洁描述}

### 方案
{具体方案，要点列出}

- {要点1}
- {要点2}
- {要点3}

### 资源需求
| 资源 | 数量 | 说明 |
|------|------|------|
| ... | ... | ... |

### 风险
- {风险1}: {应对}

### 结论建议
- 建议：{采纳/修改/否决}
```

### 会中讨论

**TeamLead广播**：
```
【一页纸提案】讨论
各提案已汇总
请针对提案讨论
时间：{分钟}
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 提案反馈

### 对提案{Agent X}的看法
- 支持/顾虑/反对：{立场}
- 原因：{理由}

### 综合/调整建议
- {建议}
```

### 会后总结

**TeamLead汇总**：
```markdown
## 一页纸提案结果

### 采纳提案
- {提案}

### 修改建议
- {修改内容}

### 行动项
| 序号 | Who | What | When |
|------|-----|------|------|
| ... | ... | ... | ... |
```

## 适用场景

- 节奏较快的决策
- 仍希望有结构输入
- 非重大但需有依据的决策
- 比口头汇报强

## 注意事项

- 必须是一页内，简洁
- 结构完整，不缺部分
- 比六页纸轻，但比口头汇报强
- 适合快速评审
