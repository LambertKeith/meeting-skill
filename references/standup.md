# 站会 (Daily Stand-up / Scrum Daily)

## 方法概述

高频同步机制，经典三问题：昨天做了什么、今天要做什么、有什么阻碍。

## 核心目的

- 高频同步状态
- 及时暴露阻塞项
- 不在会上展开长篇讨论

## 三问题结构

| 问题 | 内容 | Agent输出 |
|------|------|-----------|
| 1 | 昨天做了什么 | 已完成任务 |
| 2 | 今天要做什么 | 今日计划 |
| 3 | 有什么阻碍 | 阻塞项 |

## Agent适配流程

### 标准站会模板

**TeamLead广播**：
```
【站会】时间盒{分钟}
任务：每人汇报三问题
规则：简洁汇报，不展开讨论
阻塞项会后单独处理
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 站会汇报

### 昨天完成
- {任务1}
- {任务2}

### 今天计划
- {任务1}
- {任务2}

### 阻塞项
- {阻塞1}: {描述}
- 无阻塞
```

### 站会汇总

**TeamLead汇总**：
```markdown
## 站会汇总

### 今日计划任务
| Agent | 任务 | 预期完成 |
|-------|------|----------|
| A | ... | ... |
| B | ... | ... |

### 阻塞项
| 阻塞 | 影响Agent | 处理安排 |
|------|-----------|----------|
| ... | Agent A | 会后处理 |

### 无阻塞项
- {Agent列表}
```

### 阻塞项后续处理

```
会后单独协调阻塞项，不在站会展开讨论
记录到 action-items.md
```

## 适用场景

- 任务节奏快的团队
- 协作依赖强的团队
- 每日/定期同步
- 项目推进阶段

## 注意事项

- 时间盒严格控制（通常15分钟内）
- 不展开讨论，阻塞项会后处理
- 核心是暴露问题，不是解决问题
- 可结合W3形成行动项
