# 点投票法 (Dot Voting)

## 方法概述

快速收敛方法，每人拿若干票投给重要选项，得票高者进入下一轮或优先处理。

## 基本规则

- 每人获得{N}张票
- 可投票数可分配给多个选项
- 也可全部投给一个选项（表示强烈支持）
- 按得票数排序

## Agent适配流程

### Phase 1 - 确定候选选项

**输出模板**：
```markdown
## 候选选项

| 序号 | 选项 | 简介 |
|------|------|------|
| A | ... | ... |
| B | ... | ... |
| C | ... | ... |
| D | ... | ... |
| E | ... | ... |
```

### Phase 2 - 投票

**TeamLead广播**：
```
【点投票】
票数：每人{N}票
规则：可分散投票或集中投票
请投票并说明理由。
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 投票

### 投票分配
| 选项 | 票数 | 理由 |
|------|------|------|
| A | {票数} | {为何重要} |
| B | {票数} | {为何重要} |
| ... | ... | ... |

### 理由说明
{投票策略说明}
```

### Phase 3 - 结果汇总

**TeamLead汇总**：
```markdown
## 点投票结果

| 选项 | 得票 | 排名 | Agent投票分布 |
|------|------|------|---------------|
| A | {总数} | 1 | A:{票},B:{票}... |
| B | {总数} | 2 | ... |
| ... | ... | ... | ... |

### 决定
- 进入下一轮: {选项A、B}
- 优先处理: {选项A}
- 挂起: {选项C、D、E}
```

## 适用场景

- 多议题优先级排序
- 多方案初步筛选
- 创意收敛
- 不适合替代深度决策

## 注意事项

- 高票不等于可行，后续还需可行性分析
- 高票不等于风险最小
- 适合"初步排序"，不适合"最终决策"
- 可配合时间盒快速完成
