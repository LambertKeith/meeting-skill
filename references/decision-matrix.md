# 决策矩阵

## 方法概述

多标准决策法，在多个方案间做理性比较。确定评价标准 → 赋权重 → 对方案评分 → 比较总分。

## Agent适配流程

### Phase 1 - 确定评价标准

**TeamLead广播**：
```
【环节1】确定评价标准
任务：列出决策需要考虑的评价维度
输出格式：
| 标准名称 | 权重(1-5) | 说明 |
```

**Participant输出**：
```markdown
## {Agent名称} - 建议的评价标准

| 标准 | 权重 | 说明 |
|------|------|------|
| ... | ... | ... |
```

### Phase 2 - 确定候选方案

**输出模板**：
```markdown
## 候选方案列表

| 序号 | 方案名称 | 简介 |
|------|----------|------|
| A | ... | ... |
| B | ... | ... |
| C | ... | ... |
```

### Phase 3 - 方案评分

**TeamLead广播**：
```
【环节3】方案评分
任务：对每个方案在各标准下评分(1-10分)
输出格式：完整的评分矩阵
```

**Participant输出模板**：
```markdown
## {Agent名称} - 方案评分

| 标准(权重) | 方案A | 方案B | 方案C |
|-------------|-------|-------|-------|
| 标准1(5) | 8 | 6 | 7 |
| 标准2(3) | 5 | 9 | 4 |
| ... | ... | ... | ... |
| **加权总分** | **...** | **...** | **...** |

### 推荐方案
- {方案X}，理由：...
```

### Phase 4 - 综合决策

**TeamLead汇总**：
```markdown
## 综合评分结果

| 方案 | 平均分 | 最高分Agent | 最低分Agent |
|------|--------|-------------|-------------|
| A | ... | ... | ... |
| B | ... | ... | ... |

## 最终决策
- 选择方案：{方案}
- 决策依据：...
- DecisionMaker确认：...
```

## 示例

**选择技术方案的评价标准**：

| 标准 | 权重 | 说明 |
|------|------|------|
| 开发成本 | 5 | 人力和时间投入 |
| 上线速度 | 4 | 能多快交付 |
| 可维护性 | 3 | 长期维护难度 |
| 扩展性 | 3 | 未来扩展能力 |
| 风险程度 | 4 | 技术风险和业务风险 |

## 适用场景

- 技术方案选择
- 工具选型
- 供应商选择
- 资源分配决策

## 注意事项

- 权重和评分本身带有主观性，需团队共识
- 分数只是辅助，最终决策权在Approver
- 评分差异大时需讨论原因
