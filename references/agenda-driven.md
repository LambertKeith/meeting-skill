# 议程驱动会议法

## 方法概述

最普遍、实用的会议方法。核心是按议程推进，有明确的会议控制机制。

## 议程设计原则

**错误示范**：
- ❌ "讨论产品上线"
- ❌ "聊聊技术方案"

**正确示范**：
- ✅ "确认上线范围，产出功能边界结论"
- ✅ "确认风险清单，产出上线阻断项"
- ✅ "确认负责人和时间，产出行动清单"

## 议程模板

```markdown
# 会议议程

## 会议信息
- 会议主题: {主题}
- 会议目标: {明确的目标}
- 会议类型: {类型}
- 预期时长: {分钟}

## 议题列表

| 序号 | 议题 | 时间盒 | 预期产出 | 方法 |
|------|------|--------|----------|------|
| 1 | {议题名称} | {分钟} | {具体产出} | {方法} |
| 2 | {议题名称} | {分钟} | {具体产出} | {方法} |
| 3 | {议题名称} | {分钟} | {具体产出} | {方法} |

## 角色分配
- Driver: {推进者}
- Approver: {决策者}
- Contributors: {参与者}

## 输入材料
- {材料1}
- {材料2}

## 规则说明
- {发言规则}
- {时间规则}
```

## Agent适配流程

### 会前

**TeamLead操作**：
```
1. 撰写议程 → Write agenda.md
2. 广播议程 → SendMessage to "*"
3. 确认参会 → 收集Contributors确认
```

### 会中

**推进模板**：
```
【议题{N}】{议题名称}
时间盒: {分钟}
目标产出: {预期产出}
当前方法: {使用的方法}

请各Contributors在{时间}内完成讨论。
```

**环节结束模板**：
```
【议题{N}结束】
结论: {得出的结论}
产出: {具体产出物}
下一步: {进入议题N+1 / 会议结束}
```

### 会后

```
1. 汇总各议题结论
2. 撰写纪要 → Write minutes.md
3. 撰写行动项 → Write action-items.md
```

## 适用场景

- 几乎所有会议的基础框架
- 日常周会/项目推进会
- 方案评审会
- 问题分析会

## 注意事项

- 议程是"产出列表"，不是"话题列表"
- 每个议题必须有明确产出
- 时间盒严格执行
- Driver需控制节奏，不让次要议题占用过多时间
