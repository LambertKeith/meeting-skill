# 决策树讨论法

## 方法概述

把"如果A成立，则走X；如果A不成立，则走Y"的逻辑明确画出，适合不确定性较高的问题。

## 核心价值

不逼团队给出单一答案，而是把不同条件下的行动路径提前准备好。

## Agent适配流程

### Phase 1 - 确定决策点

**TeamLead广播**：
```
【决策树】{决策主题}
任务：梳理决策的关键条件节点和分支路径
输出：决策树图（文本形式）
```

**Contributor输出模板**：
```markdown
## {Agent名称} - 决策树建议

### 主决策点
{核心决策：要不要...}

### 条件节点

节点1: {条件A}
- 条件成立 → 路径X：{行动}
- 条件不成立 → 节点2

节点2: {条件B}
- 条件成立 → 路径Y：{行动}
- 条件不成立 → 路径Z：{行动}

### 决策树图示
```
{决策点}
├── 条件A成立 → {行动X}
│   ├── 条件B成立 → {行动X1}
│   └── 条件B不成立 → {行动X2}
└── 条件A不成立 → {行动Y}
```

### 需验证的条件
| 条件 | 验证方法 | 时间 |
|------|----------|------|
| A | {如何验证} | {何时} |
| B | ... | ... |
```

### Phase 2 - 汇总决策树

**TeamLead汇总**：
```markdown
## 综合决策树

### 决策主题
{核心决策}

### 决策树
```
{汇总后的完整决策树}
```

### 路径说明
| 路径 | 条件组合 | 行动 | 风险 |
|------|----------|------|------|
| X | A成立 | ... | ... |
| Y | A不成立+B成立 | ... | ... |
| Z | A不成立+B不成立 | ... | ... |

### Approver确认
- 决策树认可: ✅/❌
- 当前条件判断: {A的条件判断结果}
- 执行路径: {路径X/Y/Z}
```

## 适用场景

- 市场进入决策
- 项目立项决策
- 预算调整决策
- 是否继续投入某方案
- 高不确定性问题

## 注意事项

- 条件要可验证
- 每条路径都要有明确行动
- 不确定时准备多条路径
- 适合"准备选项"，而非"立即拍板"
