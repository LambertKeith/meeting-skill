# 会议效果验证框架

## 概述

本文档定义会议效果验证的框架、维度和方法，确保会议按照规范执行，产出符合质量标准。

---

## 验证目标

### 核心验证目标

1. **流程正确性**: 会议是否按议程推进
2. **角色履行度**: 各角色职责是否落实
3. **输出完整性**: 必须文档是否产出
4. **输出质量**: 内容是否符合模板要求
5. **方法一致性**: 使用方法是否正确执行

---

## 验证维度

### 1. 流程正确性

| 检查项 | 验证方式 | 通过标准 |
|--------|----------|----------|
| 议程环节完整执行 | 检查process-log | 所有议程环节有执行记录 |
| 时间盒遵守 | 检查时间偏差 | 偏差<20% |
| 消息格式正确 | 检查SendMessage记录 | 格式符合模板 |
| 环节顺序正确 | 检查process-log | 顺序与议程一致 |

### 2. 角色履行度

| 角色 | 检查项 | 验证方式 | 通过标准 |
|------|--------|----------|----------|
| TeamLead | 议程创建 | 检查agenda.md存在 | ✅ |
| TeamLead | 角色分配 | 检查roles.md存在 | ✅ |
| TeamLead | 环节广播 | 检查SendMessage记录 | 每环节有广播 |
| TeamLead | 汇总输出 | 检查phase输出 | 每环节有汇总 |
| TeamLead | 纪要撰写 | 检查minutes.md存在 | ✅ |
| DecisionMaker | 决议确认 | 检查minutes.md | 决议有DecisionMaker确认 |
| DecisionMaker | 分歧裁决 | 检查process-log | 分歧有裁决记录 |
| Participant | 环节输出 | 检查phase输出 | 每环节有输出 |

### 3. 输出完整性

| 文件 | 必需性 | 验证方式 | 通过标准 |
|------|--------|----------|----------|
| agenda.md | 必须 | 检查存在 | ✅ |
| roles.md | 必须 | 检查存在 | ✅ |
| phases/ | 必须 | 检查目录存在且有文件 | ✅ |
| minutes.md | 必须 | 检查存在 | ✅ |
| action-items.md | 必须 | 检查存在 | ✅ |
| process-log.md | 必须 | 检查存在 | ✅ |

### 4. 输出质量

| 检查项 | 验证方式 | 通过标准 |
|--------|----------|----------|
| 纪要格式正确 | 检查minutes.md结构 | 符合minutes-template.md |
| 行动项W3格式 | 检查action-items.md | 每项有Who/What/When |
| 决议有理由 | 检查minutes.md决议部分 | 每项决议有理由 |
| 共识已记录 | 检查minutes.md共识部分 | 共识点已列出 |
| 待决已记录 | 检查minutes.md待决部分 | 待决问题已列出 |

### 5. 方法一致性

| 方法 | 检查项 | 验证方式 | 通过标准 |
|------|--------|----------|----------|
| ORID | O/R/I/D四个环节 | 检查phases/ | 四个环节输出存在 |
| 决策矩阵 | 标准/方案/评分/决策四个环节 | 检查phases/ | 四个环节输出存在 |
| 六顶思考帽 | 帽子顺序正确 | 检查process-log | 顺序符合推荐 |
| 头脑风暴 | 发散/筛选/深化环节 | 检查phases/ | 环节输出存在 |
| 敏捷回顾 | Stop/Start/Continue | 检查phase输出 | 三类输出存在 |

## 验证方法

### 自动检查

通过检查文件存在性和结构进行自动验证：

```markdown
## 自动检查清单

| 检查项 | 检查方式 | 结果 |
|--------|----------|------|
| 文件完整性 | 检查6个必需文件存在 | ✅/❌ |
| 目录结构 | 检查phases/目录存在 | ✅/❌ |
| 纪要结构 | 检查minutes.md章节结构 | ✅/❌ |
| 行动项格式 | 检查action-items.md表格结构 | ✅/❌ |
```

### 人工检查

需要人工判断质量的检查项：

```markdown
## 人工检查清单

| 检查项 | 判断标准 | 评分(1-5) |
|--------|----------|-----------|
| 纪要内容质量 | 决议清晰、理由充分 | {分数} |
| 行动项可执行性 | 具体、可操作 | {分数} |
| 共识形成质量 | 有共识、分歧处理得当 | {分数} |
| 方法使用正确性 | 按方法规则执行 | {分数} |
```

---

## 评分标准

### 总体评分

| 分数 | 等级 | 说明 |
|------|------|------|
| 90-100 | 优秀 | 所有检查项通过，质量高 |
| 70-89 | 良好 | 核心检查项通过，有少量问题 |
| 60-69 | 合格 | 必需检查项通过，有较多问题 |
| <60 | 不合格 | 必需检查项未通过 |

### 必需检查项（不通过则不合格）

1. 6个必需文件全部存在
2. 决议有DecisionMaker确认
3. 行动项有W3格式
4. 议程环节全部执行

---

## 验证流程

```
会议结束
    ↓
自动检查文件完整性
    ↓
检查流程日志
    ↓
检查纪要结构
    ↓
检查行动项格式
    ↓
检查方法一致性
    ↓
人工检查质量
    ↓
计算总分
    ↓
输出验证报告
```
