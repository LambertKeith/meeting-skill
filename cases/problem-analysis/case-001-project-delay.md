# 案例：项目延期分析会

## 案例信息

| 项目 | 内容 |
|------|------|
| 案例编号 | case-001-project-delay |
| 会议类型 | 问题分析会 |
| 使用方法 | ORID + 5Why |
| 参会Agent数 | 3 |
| 环节数 | 5 |
| 总时长 | 40分钟 |
| 效果评分 | 4.5 |

---

## 用户触发

```
"请组织一个问题分析会，分析为什么项目延期了"
```

---

## 预期流程

### Phase 1: 会议启动（2分钟）

**TeamLead操作**:
```
1. TeamCreate(name="meeting-analysis-20260408-001")
2. Write("agenda.md")
3. Write("roles.md")
4. Agent spawn participant-1
5. Agent spawn participant-2
6. SendMessage(to="*", message=开场广播)
```

### Phase 2: ORID Phase O - 客观事实（10分钟）

**TeamLead广播**:
```
SendMessage({
  to: "*",
  summary: "环节O：客观事实收集",
  message: "【环节O】客观事实收集\n\n任务：列出所有客观事实、数据、现象\n时间盒：10分钟\n\n输出格式要求：\n## {你的Agent名称} - 客观事实\n### 观察到的事实\n1. ...\n\n请在10分钟内完成输出。"
})
```

**Participant输出**:
```markdown
## participant-1 - 客观事实

### 观察到的事实
1. 项目原定上线日期为3月15日，实际于3月18日上线
2. 延期3天
3. 测试阶段发现15个Bug，修复耗时2天
4. 开发提测延迟1天

### 相关数据
- 原计划工期: 30天
- 实际工期: 33天
- 测试Bug数: 15个
```

### Phase 3: ORID Phase R - 感受反应（5分钟）

**TeamLead广播**:
```
SendMessage({
  to: "*",
  summary: "环节R：感受反应",
  message: "【环节R】感受反应\n\n任务：表达对延期事实的感受\n时间盒：5分钟"
})
```

**Participant输出**:
```markdown
## participant-1 - 感受反应

### 对关键事实的反应
- 延期3天令我担忧，因为客户对上线时间有期待

### 担忧
- 担心影响客户信任
- 担心团队士气受影响
```

### Phase 4: ORID Phase I + 5Why - 根因分析（15分钟）

**TeamLead广播**:
```
SendMessage({
  to: "*",
  summary: "环节I：根因分析",
  message: "【环节I】根因分析\n\n任务：使用5Why分析法找出根因\n时间盒：15分钟"
})
```

**Participant输出**:
```markdown
## participant-1 - 根因分析

### 5Why分析
1. Why项目延期？→ 开发提测延迟1天，测试发现Bug修复耗时2天
2. Why开发提测延迟？→ 需求变更导致开发返工
3. Why需求变更？→ 需求澄清阶段未充分理解客户需求
4. Why未充分理解？→ 需求文档不够详细，缺少场景说明
5. Why缺少场景说明？→ 需求评审阶段业务方未参与

### 根因结论
- 根因：需求评审阶段业务方未参与，导致需求理解偏差
```

### Phase 5: ORID Phase D - 决策行动（8分钟）

**TeamLead广播**:
```
SendMessage({
  to: "*",
  summary: "环节D：决策行动",
  message: "【环节D】决策行动\n\n任务：基于根因分析，制定行动项\n时间盒：8分钟"
})
```

**Participant输出**:
```markdown
## participant-1 - 决策行动

### 结论
- 需求评审阶段需引入业务方参与机制

### 建议行动项
| 序号 | 行动 | 负责人建议 | 时间建议 |
|------|------|------------|----------|
| 1 | 需求评审增加业务方确认环节 | 产品经理 | 4月15日前 |
| 2 | 建立需求变更流程 | 项目经理 | 4月20日前 |
| 3 | 增加风险缓冲时间 | 项目经理 | 下次迭代 |
```
